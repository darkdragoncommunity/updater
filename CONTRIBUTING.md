# Contributing to the Game Launcher

Thank you for your interest in contributing! We welcome all improvements, from bug fixes to new features.

## Development Workflow

1. **Fork the repository** and create your branch from `main`.
2. **Setup the environment**: Ensure you have C++17, CMake, and the necessary dependencies (libcurl, OpenGL, GLFW).
3. **Make your changes**: Follow the existing coding style and ensure your code is well-documented.
4. **Test thoroughly**: Verify that the launcher still works with your changes.
5. **Submit a Pull Request**: Provide a clear description of the changes and why they are needed.

## Coding Standards

- Use PascalCase for class names and methods.
- Use camelCase for variables.
- Keep functions small and focused on a single task.
- Add comments for complex logic.

## Security

If you find a security vulnerability, please do NOT open a public issue. See our [Security Policy](SECURITY.md) for reporting instructions.
